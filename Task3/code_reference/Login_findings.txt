Login.java – Security Findings

File:
Login.java

Location of Vulnerable Logic:
generateSessionToken() method

Relevant Code Behavior:
- The application generates a session authentication token using `java.util.Random`.
- The token is created in the method `generateSessionToken()` and returned as a 16‑character string.
- The generated value is stored in SharedPreferences using the key `sessionToken` via the method `createSession()`.

Code References:
1. Random generator import
   import java.util.Random;

2. Token generation
   private String generateSessionToken() {
       Random random = new Random();
       StringBuilder sb = new StringBuilder(16);
       for (int i = 0; i < 16; i++) {
           sb.append("ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789"
               .charAt(random.nextInt(62)));
       }
       return sb.toString();
   }

3. Session token storage
   public void createSession() {
       this.editor.putString(KEY_SESSION_TOKEN, generateSessionToken());
       this.editor.apply();
   }

Security Issue:
`java.util.Random` is a predictable pseudo‑random number generator and is not suitable for generating security‑sensitive tokens such as authentication or session identifiers.

Security Impact:
An attacker who understands the token generation logic could attempt to predict or brute‑force valid session tokens, potentially allowing session hijacking or impersonation of authenticated users.

Recommended Fix:
Replace `java.util.Random` with `java.security.SecureRandom`, which provides a cryptographically secure random number generator appropriate for authentication tokens.
